# 基于异构图注意力网络与指针网络的飞机脉动装配线智能调度系统 (HB-GAT-PN) - v5.0

本项目是一个融合了**深度强化学习 (PPO)**、**异构图注意力机制 (Heterogeneous GAT)** 与 **自回归指针网络 (Pointer Network)** 的工业级调度仿真与智能排产系统。专为解决“工序-站位-工人”三层高度耦合、包含严格拓扑前置约束的飞机装配线调度难题而设计。

本项目在传统的启发式与遗传算法基线上取得了巨大的突破，最新版本 `v5.0` 具备在 3000+ 节点（包含超长时序挑战）的大型拓扑图中维持极高稳定性与收敛效果的能力，且支持了**域随机化抗干扰**和绝对无瑕疵的工业排程导出方案。

## 📖 数据集精析与拓扑构建原理 (Data Parsing & Routing)

进入系统前，最核心的是理解支撑上千节点异构网络的真实物理数据集结构，以 `3000.csv` 文件为例。系统通过 `data_loader.py` 将扁平的数据行解析为层次分明的异构有向无环图 (DAG)。

### 1. `3000.csv` 核心数据字段展示

| 字段名称       | 解析别称 / 内部映射           | 含义与作用 |
| :---           | :---                         | :--- |
| **序号**       | `internal_id`                | 数据的行标识。系统后台实际运算图结构的唯一标识符 (0-based Index)，确保与输入文件顺序锁死。 |
| **AO号**       | `task_id`                    | 飞机装配大纲编号。**它是划分“根-组-叶”层级的核心启发式特征**。 |
| **类型**       | `skill_type`                 | 任务所需工种 (0~9 的整数，映射到工人的具体技能表)。 |
| **紧前工序AO号**| `predecessors`               | 指定必须在本任务开启前 100% 竣工的先决工序，支持多工序依赖 (逗号分隔)。 |
| **需求人数**   | `demand_workers`             | 该工序所需并行配置的工人额度。若不满编，该节点将被视为 Deadlock（系统锁死惩罚）。 |
| **加工时间/h** | `duration`                   | 物理层面的标准制程节点耗时。在 `eps=0` 的贪婪验证中直接套用。 |
| **限定站位**   | `fixed_station`              | 如果有值，则该工序在 Station 选择阶段会被强行判定，其它物理站位 Mask 为无穷小。 |

### 2. `data_loader.py` 图网络层级解析逻辑

在极其复杂的航空拼装中，任务往往是按区域块 (Level 1) 和小组件 (Level 2) 推演的。`data_loader.py` 会依据 **AO号字符串特征** 自动生成极为严密的装配流水线隐式网络结构：

1. **Root (根节点，Level 1)**：如 AO号为纯字母 `"A"`, `"B"` 等。
2. **Sub (组节点，Level 2)**：如 AO号为根节点附加横杠与数字 `"A-1"`, `"B-2"` 等。
3. **Task (真实工序，Leaf/Level 3)**：其余长串 AO号，如 `"AAQS00-0010"`。

**无缝拓扑连接规则 (隐式连边与显式连边)**：
`data_loader.py` 在加载宽表后，不仅读取显式的 `紧前工序AO号` (Rule D)，更会自动勾勒出如下流水线隐含强制流向：
- **组控子 (Rule A)**: Sub 节点连向该子组下的全部真实 Task。
- **子归源 (Rule B)**: 该被控的 Task 完成后，箭头汇聚指向该 Root 下的下一个 Sub 节点。
- **虚空跨接 (Rule C / E)**: Root 必定指向首个 Sub；上一个 Root 内最后完成的单元必定指向流水线的下一个 Root 开端。

通过这种“硬编码特征检测 + Soft连边构建”，环境基建（`Environment`）无需人员大量人工标注，即可自主生成极其强效的前置阻塞关系网络，并实时接管仿真器中的 `Ready` 可分配状态（只有前置箭头全部放行的工序才会被抛入决策池）。

## 📂 核心代码目录与架构全景

系统遵循标准的强化学习环境闭环结构：**数据加载** -> **环境仿真 (离散事件引擎)** -> **神经网络前向推断** -> **反向传播与 PPO 策略更新**。

```text
ALB_RL_Project/
│
├── configs.py             # 系统的全局控制中枢（包含针对 3000 单的长序列防崩溃极限参数）
├── data_loader.py         # 任务工序拓扑与人员信息的统一解析器（解析根/组/叶节点及边关系）
├── environment.py         # 基于 Gymnasium 规范构建的带域随机化离散事件仿真环境
├── ppo_agent.py           # 强化学习 PPO 算法代理（执行采样、GAE计算、Huber Loss容错）
├── train.py               # 模型的主训练循环 (包含 Tensorboard 极简监控)
├── evaluate.py            # 模型的标准验证入口
├── generate_schedule.py   # [v5.0 新增] 排程结果的无噪音导出管线 (生成 .csv 报表)
│
├── models/                # 神经网络的核心组件目录
│   └── hb_gat_pn.py       # 包含了从图卷积到级联自回归 (Pointer) 的核心大脑拓扑设计
│
└── utils/
    └── muon.py            # 牛顿-舒尔茨正交化的高阶动量优化器 (Muon)
```

## 📜 脚本级别深度解析

以下是对项目中每个核心脚本的类、主要方法、核心逻辑的详尽阐述：

### 1. `configs.py` (全局配置文件与极限防崩中枢)
此文件相当于系统的**中央控制台**。所有可调控的数值汇聚于此，代码其余处均通过动态获取 (`getattr(configs, ...)`) 或引入保证耦合度极低。在 `v5.0` 面对极长序列 (3300步长) 时搭载了一系列激进参数：

* **系统拓扑上限设定** (`data_file_path`, `n_j`, `n_m`, `n_w`): 特别是设定 `n_j=3300` 撑开最大的输入掩码天顶。由于数据底层采用稀疏图表示，可实现远超寻常图规模的处理上限。
* **模型超参** (`hidden_dim`, `num_gat_layers`, `dropout`): 用于定义图表征传播深度及多头注意力等基质。
* **PPO 深度抗拟合武器库**:
  * `batch_size`: 设为 16，严命低内存爆破可能。
  * `accumulation_steps` (128步): **梯度爆冷累积核心**，等效一个包含 2048 (16x128) 步的大局势视野，严防 3000 工序下的长序列前期小气泡经验过拟合（俗称：死磕局部极大值）。
  * `gamma` & `gae_lambda`: 长线折现双霸，提升到 `0.9995` 及 `0.98` 以保证几千步的长远惩罚能够如实穿越维度警示排头的鲁莽行动（视力极度远视）。

---

### 2. `environment.py` (环境与离散仿真核心)
这是整个项目的物理与时间引擎核心。通过继承 `gym.Env` 建立，它推进**离散事件仿真 (Discrete Event Simulation)**计算图结构演变。

#### 离散引擎基础机制 (`Event`)
用来包裹仿真中即将发生的事件（当前是 `TASK_FINISH`），放置入系统的优先队列进行时序排列。

#### `AirLineEnv_Graph(gym.Env)`
* **节点网络图构建 (`_calculate_cpm`)**: 执行经典的最早/最迟时刻解算，算出关键路径 (Critical Path)，并向这些可能引起后续多米诺骨牌延误的节骨眼盖上特征印记。
* **域随机化 (Domain Randomization)**: **【v5.0 新增】**。由于训练集固化，模型极易对于某些特定的工时配比发生绝对依赖死锁。在 `env.reset(randomize_duration=True)` 状态下，所有的任务工时 (`duration`) 将跟随 `dur_random_range`（如 20%的区间）产生白噪音浮动，迫使策略收敛到高兼容形态而非钻参数死胡同。
* **时序与步进系统 (`_advance_time`) & (`step`)**: 当场上可用人员耗尽或无任务放行时，引擎不空转，直接时钟跃迁 (`_advance_time`) 到下一个任务完结的时点去释放资源。`step(action)` 接受自回归产生的选人、选站、选工序元组后返回图异构特征 (`_get_observation`) 和动态掩码。

---

### 3. `ppo_agent.py` (网络结构代理控制中枢)
包裹了核心算法、梯度裁剪、记忆体、以及对于异构预测大脑剥离的核心逻辑模块。

#### 核心修正与防护措施：
1. **梯度安全剥离 (Critic Detachment)**:
   在此前由于 `global_context` 被同时共享给 Actor（执行下属）和 Critic（考评官），Critic 的负反馈很容易通过 context 回传捣毁底层的图网络特征。`v5.0` 使用了 `global_context.detach()` 防火墙，将 Critic 严格限制在顶端。
2. **Huber Loss 对抗数值崩溃 (Value Error Overflow)**: 
   由于 3000工序 跑偏会带来多达上万分的 Makespan 死锁负反馈。传统的 MSE (均方误差) 平方放大后会导致数亿级的梯度冲入 Critic，触发 PPO 全局 `clip_grad_norm_` 将原本健康的 Actor 梯度拉到底造成全网瘫痪变“瞎子”。当前使用了 `Huber Loss` 保障数十级以上的误差全部线性传导。
3. **PPO Surrogate 核心结构**:
   内置蒙特卡罗 fallback、GAE 计算、熵最大化以对抗环境初期搜索贫瘠问题。采用 AdamW + Muon 混合动力推进。

---

### 4. `train.py` & `evaluate.py` (主回路控制与快速推演)
#### `Memory`
用于一次性托管轨迹（Trajectory）的所有信息。
#### `train()`
系统训练的循环体，囊括对环境时序状态（`done`, `deadlock`）拦截。引入 TensorBoard 写出 `Loss/Policy`, `Eval/Makespan`, `Eval/Balance_Std`。在规定轮次收集满样本后通知 Agent 开火。
#### `evaluate_model()`
测试主框架，开启 `env.reset(randomize_duration=False)` 后，屏蔽所有的 `categorical.sample()` 转为极端的确定性 `Argmax` 以探寻策略天顶上限。

---

### 5. `generate_schedule.py` (排程报告提取) 
**【v5.0 实用组件】**
为弥补之前不可视化的黑箱问题提供，能够自动在 `models` 目录下寻找最近修改的检查点权重（如：`ALB_PPO_Best10000.pth`）。然后关闭温度采样（纯贪心）跑一轮完全确定性的 3000 单无干扰真题。
* **功能输出**: 完成推演后，脚本会抽取 `env.completed_tasks` 后端引擎内存留的任务记录。剥离虚空和根组节点，利用 `data_loader` 反查字典将原始 `AO号` 兑现并附上最终的分配人员与发生时间表，重构成 `final_schedule.csv` 以用于现场的排产投屏。

---

### 6. `models/hb_gat_pn.py` (图神经网络架构核心)
模型大脑中枢，基于 `torch_geometric.nn.HeteroConv` 和 `GATConv` 构建图空间推理。
1. **深度信息汇聚**: 利用异构图特质，将 `Task`, `Worker`, `Station` 三维本体通过连边特征互刷，将全流水线当前谁是卡脖子点的信息聚合进入 `global_context`。
2. **级联自回归指针网络 (Pointer Network)**: 输出并不是常规分类的 Softmax 单层预测，而是一场级联询问：
   - 第一级 `TaskPointer`: 在当前可选节点群中寻找最优的任务。
   - 第二级 `StationSelector`: 结合当前选中任务的情报找个空闲工作站。
   - 第三级 `WorkerPointer`: 以需求人数（`demand`）为基准启动一个动态长度的 RNN `for` 循环，逐一挑选合格工人放入阵列中，选出即动态封闭 Mask（防止克隆人问题），直到筹集人手。

## 🌟 学术包装与系统亮点 (Academic Highlights)
**【v5.0 突破】硬编制定岗/物理防线约束 (Rigid Station-Binding Constraint)**  
不同于传统调度文献中假设劳动力可无成本、无视物理距离在全局自由流动的理想化设定，本文针对大飞机脉动线的物理空间隔离特性，创新性地提出了硬编制定岗约束。所有的工人在调度初期作为“全线机动状态”（Lock=0）存在，而一旦被自回归解码器选中并落地指派至某一特定站位，其物理归属位置将被永久锁定（Lock=S_act）。通过在强化学习自回归解码阶段（Pointer Network 内部）以及环境步骤演化中实施极其严密的硬性掩码隔离（Hard Masking），系统从底层彻底消除了不切实际的跨工位调度行为，使得生成的排程策略 100% 严丝合缝地满足现代车间的班组物理管理要求。
