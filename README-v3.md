# 基于分层强化学习与软约束效率矩阵的飞机脉动线智能调度系统 (HB-GAT-PN) - v6.0 / v7.0

本项目是一个融合了**深度分层强化学习 (HRL)**、**异构图注意力机制 (Heterogeneous GAT)** 与 **自回归指针网络 (Pointer Network)** 的工业级调度仿真与智能排产系统。专为解决“工序-站位-工人”三层高度耦合、包含严格拓扑前置约束的飞机装配线（尤其是 3000 单级的超大尺度）调度难题而设计。

在传统的启发式与单体强化学习基线上取得了巨大的突破，最新架构 `v7.0` 具备在 3000+ 节点大型拓扑图中完全摧毁“拓扑卡脖子与维度吸盘”死锁的能力，且支持了**软约束技能惩罚**和**发牌定岗双轨架构**。

---

## 📖 数据集精析与拓扑构建原理 (Data Parsing & Routing)

进入系统前，最核心的是理解支撑上千节点异构网络的真实物理数据集结构，以 `3000.csv` 文件为例。系统通过 `data_loader.py` 将扁平的数据行解析为层次分明的异构有向无环图 (DAG)。

### 1. 核心数据字段展示

| 字段名称       | 解析别称 / 内部映射           | 含义与作用 |
| :---           | :---                         | :--- |
| **序号**       | `internal_id`                | 数据的行标识。系统后台实际运算图结构的唯一标识符 (0-based Index)。 |
| **AO号**       | `task_id`                    | 飞机装配大纲编号。**它是划分“根-组-叶”层级的核心启发式特征**。 |
| **类型**       | `skill_type`                 | 任务所需工种 (0~9 的整数，映射到工人的具体技能表)。 |
| **紧前工序AO号**| `predecessors`               | 指定必须在本任务开启前 100% 竣工的先决工序，支持多工序依赖。 |
| **需求人数**   | `demand_workers`             | 该工序所需并行配置的工人额度。 |
| **加工时间/h** | `duration`                   | 物理层面的标准制程节点基准耗时。 |

### 2. 图网络层级解析逻辑

在极其复杂的航空拼装中，任务往往是按区域块 (Level 1) 和小组件 (Level 2) 推演的。`data_loader.py` 依据 **AO号字符串特征** 自动生成流水线隐式网络结构：
- **Root (层级1)**: 纯字母如 "A", "B"
- **Sub (层级2)**: 字母带数字如 "A-1"
- **Task (真实叶子节点)**: 长字符如 "AAQS00-0010"

系统自动勾勒强软流向，无需人工干预即可严丝合缝地在引擎端进行事件阻塞与激活计算。

## 💥 【v7.0 重磅架构颠覆】

针对在高达 3000 道工序中遇到的 “RL 局部视野陷阱”与“前排站位无情虹吸引发的全局死锁” 问题，目前系统经历了前所未有的大重构：

### 1. 软效率矩阵与技能融合 (Soft Efficiency Matrix)
- 废弃了“无对口技能就不准被选拔”的一招切硬掩码，全面释放了网络选择动作的自由度。
- 引入 `configs.soft_skill_penalty`（如 0.1）。RL 在选人阶段可以跨界混搭，但这会导致小组总合成效率暴跌，导致单一任务工时扩大 10 倍以上。这确立了极其明确的进度本位驱动力，强迫 PPO 自己学会如何找大牛，而不是依赖规则的襁褓。

### 2. 分层强化学习发牌员 (HRL Allocator Agent)
- 单靠 `Temp=0.5` 与否来决断死锁是一种极其不稳定的现象。我们直接引入了基于控制宏观时间戳周期的 **【Allocator 元智能体】** 模型！
- 系统在 `configs.allocation_period` 的驱动下，按特定周期将几百名全厂机动人员强行物理打散、下放分配进入各个固定站位。
- 这个操作等同于一座几公里长的物理流水线，S8 的驻军绝对不可能被极其遥远的 S1 抢走，从物理大世界上彻底断绝了死锁蔓延的基础！

### 3. 去伪存真：只争朝夕 (Terminal Real-Makespan Reward)
- 去除了每一步繁杂冗余的（Blocking, Efficiency）密集群体惩罚。
- 改为极其纯粹的**“做对任务奖 1 点进度，最后一把算真实下班打卡墙上时间的总账”**！这是应对大工业调度最具宏观视野的评价体系。

## 📜 关键调优与防护墙机制回顾

1. **梯度安全剥离 (Critic Detachment)**: 建立防火墙隔离 Context 回传崩溃。
2. **Huber Loss 兜底**: 对抗因超长时间线带来的数十万级的值误差崩盘，严防 PPO "瞎眼"。
3. **梯度爆炸前聚集 (Accumulation Steps)**: 采用 128 步收集的极远视策略避免 3000 单上的早熟近视病！
4. **混合型掩码 (Hybrid Masking) 保留**:
   - `Hard Station Limits`: 限制站位的人员过度超载容量阈值 (`0.6`)。
   - `Workload Limit`: 防波堤拦截，防止某一站位独揽大权。

## 🚀 起步指南

运行环境需要包含 `PyTorch (CUDA 13.1)` 及相关的图工具集（`torch_geometric`）。推荐 `rag_env`。

```bash
# 激活环境
conda activate rag_env

# 启动融合了 HRL 分区宏观调配与软效率驱动的大模型训练之旅：
python train.py

# 在评估中途发现 Makespan 达到新的奇迹巅峰时，导出车间落地版报表：
python generate_schedule.py
```
